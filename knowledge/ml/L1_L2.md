# L1 vs L2 Regularization

## 1. Overview

Regularization prevents **overfitting** by penalizing large weights.

We modify the original loss function:

- **L1 (Lasso):** penalizes absolute values
- **L2 (Ridge):** penalizes squared values

---

## 2. Mathematical Formulation

Given base loss \( \mathcal{L}(w) \):

### L1 Regularization
$\mathcal{L}_{L1} = \mathcal{L}(w) + \lambda \sum |w_i|$

### L2 Regularization
$\mathcal{L}_{L2} = \mathcal{L}(w) + \lambda \sum w_i^2$

- \( $\lambda$ \): regularization strength  
- Larger \( $\lambda$ \) → stronger constraint → simpler model  

---

## 3. Geometry Intuition (Why Diamond vs Circle)

### Constraint Form

- L1: \( $\sum |w_i| \leq t$ \) → **diamond (L1 ball)**
- L2: \( $\sum w_i^2 \leq t$ \) → **circle (L2 ball)**

### Optimization View

- Loss surface = ellipses  
- Solution = first point where ellipse touches constraint region  

### Key Insight

- L1 has **sharp corners (axes-aligned)**  
- L2 is **smooth everywhere**

👉 Because corners align with axes → L1 often hits exactly on axes → weights become **0**

---

## 4. Sparsity Behavior

| Property | L1 | L2 |
|--------|----|----|
| Produces exact zero weights | ✅ Yes | ❌ No |
| Feature selection | ✅ Yes | ❌ No |
| Dense model | ❌ | ✅ |

### Core Idea
- L1 = **sparse solution**
- L2 = **distributed solution**

---

## 5. Gradient Behavior (Critical Difference) <- Why L1 can nullify some features

### L2 Gradient
$\frac{\partial}{\partial w} (\lambda w^2) = 2\lambda w$

- Smooth and continuous  
- Shrinks proportionally to size  
- Small weights → small gradient  

---

### L1 Gradient
$
\frac{\partial}{\partial w} (\lambda |w|) =
\begin{cases}
\lambda & w > 0 \\
-\lambda & w < 0 \\
\text{undefined} & w = 0
\end{cases}
$

- Constant magnitude  
- Always pushes toward zero  
- Non-differentiable at 0  

👉 This creates **hard thresholding → exact zeros**

---

## 6. Shrinking Dynamics

### L2 (Soft Shrinkage)
- Big weights shrink more  
- Small weights shrink less  
- Never exactly zero  

### L1 (Hard Shrinkage)
- Same force regardless of size  
- Small weights → collapse to 0  
- Large weights survive  

---

## 7. Correlated Features Behavior

### L2
- Splits weight across correlated features  
- Stable but less interpretable  

### L1
- Selects one feature, drops others  
- Unstable when features are highly correlated  

👉 Important weakness of L1

---

## 8. Bayesian Interpretation

- L2 → Gaussian prior  
$w \sim \mathcal{N}(0, \sigma^2)$

- L1 → Laplace prior  
$w \sim \text{Laplace}(0, b)$

### Insight
- Gaussian → smooth decay  
- Laplace → sharp peak at zero → sparsity  

---

## 9. Optimization Differences

### L2
- Closed-form solution exists (Ridge):
$w = (X^T X + \lambda I)^{-1} X^T y$

- Easy to optimize  
- Smooth objective  

---

### L1
- No closed-form solution  
- Requires:
  - Coordinate descent
  - Subgradient methods  

- Harder optimization due to non-differentiability  

---

## 10. Elastic Net (Combination)

$\mathcal{L} = \mathcal{L}(w) + \lambda_1 \sum |w_i| + \lambda_2 \sum w_i^2$

- Combines:
  - L1 → sparsity
  - L2 → stability  

👉 Fixes L1 instability with correlated features  

---

## 11. Effect of Feature Scaling (Very Important)

- L1 and L2 are **scale-sensitive**
- Always normalize features before applying  

Otherwise:
- Features with larger scale dominate penalty  

---

## 12. Intuition Summary

- **L1 = "kill weak features"**
- **L2 = "shrink everything smoothly"**

- L1 → sparse + interpretable  
- L2 → stable + generalizable  

---

## 13. Common Pitfalls

### ❌ L2 does feature selection
→ False, it only shrinks weights

### ❌ L1 is always better
→ Fails with correlated features

### ❌ Ignoring scaling
→ Leads to incorrect regularization behavior

---

## 14. Linear Algebra Perspective

- L2 minimizes **Euclidean norm (L2 norm)**  
- L1 minimizes **Manhattan norm (L1 norm)**  

### Consequence
- Different geometry → different solutions  

---

## 15. Quick Mental Model

| Concept | L1 | L2 |
|--------|----|----|
| Shape | Diamond | Circle |
| Behavior | Sparse | Smooth |
| Effect | Selection | Shrinkage |
| Stability | Low | High |

---

## 16. Key Takeaway

- L1 works because of **non-smooth geometry + constant gradient**
- L2 works because of **smooth geometry + proportional shrinkage**

👉 The shape of the constraint directly determines the solution behavior
